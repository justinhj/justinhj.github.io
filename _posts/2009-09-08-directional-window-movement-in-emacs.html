--- layout: post title: Directional window movement in emacs date: '2009-09-08T14:12:00.001-07:00' author: Justin tags: [emacs] modified_time: '2009-09-08T14:12:54.886-07:00' blogger_id: tag:blogger.com,1999:blog-4777243148323391813.post-8584530707656079403 blogger_orig_url: http://justinsboringpage.blogspot.com/2009/09/directional-window-movement-in-emacs.html --- One thing I sometimes miss about the text editor Brief, which I stopped using a decade ago, is being able to move between windows easily by pressing a function key and then an arrow key to switch to a window in that direction.

In emacs I only knew how to move between windows using `other-window', mapped to C-x o and that only cycles around the windows, and if you have a few windows open and possibly you're also editing the mini-buffer, that's quite a round trip.

I just learned about `M-x windmove-default-keybindings', which binds a set of commands that do exactly the kind of directional movement I'm talking about (`windmove-right', `windmove-up' and so on) to the cursor keys (with shift key held).
